# Simple planning requests

The deliverable is a small change in the workspace: typically one new file
and maybe a run to verify it. The plan should be short (2-4 steps) and need
little or no exploration first. Plan steps describe requirements in prose;
the executor writes the actual code.

## Console calculator (the classic)

```
@devteam create a python console calculator in calculator.py: show a menu with addition, subtraction, multiplication, division and exit; ask for two numbers, print the result, and handle division by zero with a friendly message instead of crashing
```

## Number guessing game

```
@devteam create guess.py: pick a random number between 1 and 100 and let me guess it in the console, answering "higher" or "lower" until I get it, then print how many tries it took
```

## Tiny utility script

```
@devteam create a script wordcount.py that takes a file path as a command line argument and prints the number of lines, words, and characters in it
```

## To-do list in one file

```
@devteam create todo.py: a console to-do list where I can add a task, list tasks with numbers, and mark one done; keep the tasks in a tasks.json file next to the script
```

## Create and run

```
@devteam create fizzbuzz.py that prints fizzbuzz from 1 to 50, then run it and show me the output
```
